## Как устроен сам сайт

Минималистичный лендинг с WebGL-героем: неоновый куб из частиц, поделённый на две кликабельные половины («Обо мне» / «Решения»), которые открывают тёмные стеклянные панели. Прокрутка морфит куб в надпись и дальше в контент. Заявка с формы летит в БД и мне в Telegram.

### Стек

- **Next.js 15** (App Router) + **React 19** + **TypeScript**
- **three.js** — куб и частицы (грузится лениво, отдельным чанком)
- **CSS Modules** — без UI-фреймворков
- **Drizzle ORM + libSQL / Turso** — хранение заявок
- **Telegram Bot API** + **Resend** — доставка уведомлений
- **Zod** — валидация формы; rate-limit + honeypot + хеш IP

### Структура

- `concepts/ink-swarm-v1/` — текущий лендинг (`/` экспортирует его)
  - `Swarm.tsx` — WebGL-куб на raw three.js
  - `InkSwarmLanding.tsx` — разметка, панели, секции
  - `Graphics.tsx` — иконки и нейро-сеть
- `app/api/leads/` — приём заявок (валидация, rate-limit, нотификации)
- `lib/` — БД, валидация, доставка уведомлений, телеграм-ассистент

### Локальный запуск

```bash
npm install
cp .env.example .env.local   # заполни нужные переменные
npm run dev
```

Переменные окружения — в [`.env.example`](.env.example). Без внешних каналов заявка всё равно сохранится в локальную SQLite (`data/leads.db`), а API вернёт `delivery_error`.

## Заявки

Форма → `POST /api/leads` → валидация (Zod) → запись в БД → уведомление в Telegram / Email. IP хешируется, есть honeypot и rate-limit.
